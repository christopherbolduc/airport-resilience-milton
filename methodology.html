Methodology

How the Airport Resilience Score was built and calculated

Analytical Approach

To evaluate airport resilience during Hurricane Milton (Oct 2024), I sourced daily flight operations and weather data across five Florida airports. The goal was to measure how well each airport recovered, not just how badly it was hit.

Key Metrics

  • Days to recovery — Time until flight volume and cancellations returned to near-normal
  • Cancellation rate — Total flight cancellations during October
  • Peak wind speed — Used to normalize for storm severity using square-root scaling

Resilience Score Formula

The final score balances operational disruption and storm intensity:

resilience_score = (
    (days_to_recovery + (pct_cancelled * 10)) /
    ((peak_wind_speed_kmh ** 0.5) / 10)
)

Higher scores indicate worse performance. The formula is designed to fairly compare recovery, adjusting for how hard each airport was hit.

Tools Used

Languages: Python, SQL

Libraries: Pandas, Matplotlib, Seaborn, SQLAlchemy, Meteostat

Environment: Jupyter Notebook, PostgreSQL (DBeaver), Git/GitHub

Data Sources

  • BTS (Bureau of Transportation Statistics) — Historical flight operations and cancellations
  • Meteostat API — Hourly and daily weather data by airport

SQL View Structure

To manage complexity, SQL views were structured in 3 layers:

  • Staging — Raw table wrappers
  • Prep — Cleaned and engineered features
  • Mart — Final analytical dataset used in notebooks and reporting

Notebook Analysis

The resilience score was calculated in a Jupyter Notebook using prepared SQL views. Charts and visuals were created with Matplotlib and Seaborn, and key metrics were exported to CSV for use in reports and dashboards.

Reproducibility

Anyone can recreate the analysis by cloning the repository, installing dependencies from requirements.txt, and running ap_resilience.ipynb with the proper source data in place.
